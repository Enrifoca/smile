Framework pillars

Four ideas, baked into the core.

Everything else is a connector or a prompt. The framework stays connector-neutral so you can rebrand it for any vertical.

01

Multi-model

Configure chat, reasoning, and OCR-capable models independently — the right model for each step.

02

Human in the loop

Read tools run freely. Write actions pause for an Accept / Refuse decision above the composer.

03

Craftable modules

Connectors contribute tools, prompt sections, auth UI, previews, and approval copy — outside the core loop.

04

Memory management

User memory is authoritative Markdown. Learned notes stay lower priority; scratchpad is per-turn.

Markdown report artifacts

Long plans and batch specs become a report card in chat — open the full document, then revise the same file in place.

report_write

Approve every write

Connector write actions surface a compact Accept / Refuse bar. Nothing leaves your machine without your say-so.

Workspace file access

Agents read, search, OCR, and write inside a folder you choose. PDFs and scans fall back to an OCR model automatically.

sandboxed

Editable memory

Durable instructions live in Markdown under .smile/memories. Edit them like a doc — the agent reads them every turn.

How it works

From a message to a safe action.

The agent loop alternates model calls and tool runs, with guards that keep multi-step work on track.

1

Assemble

Core Markdown prompts, your memory, and connector sections build the system prompt for the turn.

2

Act

The model calls core tools or connector tools — reading files, searching, drafting a report.

3

Approve

Write tools become a pending action and wait for your Accept / Refuse before anything executes.

4

Continue

Task-continuity guards nudge read→write workflows so the agent finishes instead of stopping early.

Connector-neutral by design

One core. Any domain.

The agent loop and prompts never mention a specific product. Domain behavior lives in connector modules — tools, prompts, auth, formatters. Jira ships as the reference example, not the identity.

Jira · example Your CRM Your data warehouse Your ticketing + build your own

A connector provides

▸ Tool definitions (read & write)
▸ A prompt section for its domain
▸ Auth / OAuth setup UI
▸ Action previews & approval copy
▸ Result formatting
▸ Cache invalidation rules
