Framework pillars

Four ideas, baked into the core.

Everything else is a connector or a prompt. The core stays connector-neutral so you can rebrand it for any vertical.

01

Multi-model

Configure chat, reasoning, and OCR-capable models independently — the right model for each step.

02

Human in the loop

Read tools run freely. Write actions pause for an Accept / Refuse decision above the composer.

03

Craftable modules

Connectors contribute tools, prompts, auth UI, previews, and approval copy — outside the core loop.

04

Memory management

User memory is authoritative Markdown. Learned notes stay lower priority; scratchpad is per-turn.

How it works

From a message to a safe action.

The agent loop alternates model calls and tool runs, with guards that keep multi-step work on track.

Step 1

Assemble

Core prompts, your memory, and connector sections build the system prompt for the turn.

Step 2

Act

The model calls core or connector tools — reading files, searching, drafting a report.

Step 3

Approve

Write tools become a pending action and wait for your Accept / Refuse before executing.

Step 4

Continue

Task-continuity guards nudge read→write workflows so the agent finishes the job.

Connector-neutral by design

One core. Any domain.

The agent loop and prompts never mention a specific product. Domain behavior lives in connector modules. Jira ships as the reference example, not the identity.
